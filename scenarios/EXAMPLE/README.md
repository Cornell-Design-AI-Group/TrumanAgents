# Situation Data Schema (Templates)

This folder contains **empty templates** for defining a scenario (“situation”) in the simulation.
Use these CSVs/JSON to author a new level, then populate rows as needed.

> Generated: 2025-11-03T12:56:54

---

## Files

- `scenarios.csv` — one row per scenario **situation**.
- `actors.csv` — profile data for narrative characters (victim, bully, bystanders).
- `agents.csv` — actor profiles **plus** behavior/psychometric parameters used by the policy.
- `posts.csv` — seed posts that initialize the feed for this situation.
- `replies.csv` — seed replies to the above posts.
- `objectives.csv` — win conditions and guidance shown to the player (required/optional).
- `solutions.json` — (empty) to be filled with canonical interventions per objective/category.
- `solutions_template.json` — example structure to copy/paste for `solutions.json`.

### `objectives.csv`

Defines the goals shown to the player and used by evaluation.

| column        | type        | description                                                                                                   |
| ------------- | ----------- | ------------------------------------------------------------------------------------------------------------- |
| `level`       | string/int  | Scenario level key (may be blank if global).                                                                  |
| `label`       | string      | Objective shown to the player.                                                                                |
| `category`    | string      | Canonical category that should match a `solutions.json` category (e.g., `support victim`, `rally bystander`). |
| `isRequired`  | bool        | If `true`, must be achieved to “win”.                                                                         |
| `order`       | int         | Display order in UI/checklist.                                                                                |
| `isComposite` | bool        | If `true`, objective aggregates multiple sub‑goals.                                                           |
| `required`    | string/json | Optional sub‑requirements (format project‑defined).                                                           |

### `solutions.json`

Authoritative mapping from `category` → one or more **action recipes** the system can recognize or propose.

Each item has:

- `category` (string) — must line up with `objectives.csv.category`.
- `description` (string) — plain‑language description of the intervention.
- `type` (enum) — e.g., `comment`, `dm`, `report`.
- `deltas` (int) — effect on scenario score/system health.
- `next_steps` (list) — optional scripted follow‑ups (objects with `type`, `agent`, `content`).

See `solutions_template.json` for examples.

## Authoring Flow (Suggested)

1. Create a new row in `scenarios.csv` (name + description).
2. Define characters in `actors.csv` (at minimum: victim, bully, 1–3 bystanders).
3. If an actor is simulation‑driven, add/override them in `agents.csv` and set behavior knobs.
4. Seed the feed via `posts.csv` and `replies.csv` (use `actor` usernames).
5. Write player objectives in `objectives.csv` (required + optional).
6. Add canonical `solutions.json` entries for each objective `category`.
7. Load into the game; iterate on parameters (PRS/CNT/etc.) until interactions feel right.

## Conventions

- Prefer normalized [0–1] scales for propensities.
- Use ISO 8601 timestamps for absolute time fields.
- Keep `username` stable across files to join records.
- Use consistent taxonomies for `class` and `category` across datasets.
